## Question 1
지연 렌더(FR-1)를 어떤 방식으로 구현할까요?

A) **무의존 윈도우 렌더 (권장)** — 새 라이브러리 없이, 최근 N개(기본 40개 정도) 항목만
   DOM에 마운트하고 "이전 메시지"를 청크 단위로 추가 마운트. 채팅처럼 높이가 제각각인
   콘텐츠에 안전하고, 프로젝트의 의존성 최소 기조에 맞음. 스크롤 앵커링으로 위치 튐 방지.

B) **가상 스크롤 라이브러리 도입** — `@tanstack/react-virtual` 등 런타임 의존성을 추가해
   전체 항목을 진짜 가상화. 초대형 이력에 이론상 더 강하지만, 가변 높이 측정/코드블록
   재계산 이슈와 새 의존성 비용이 있음.

## Question 2
과거(이전) 메시지를 마운트하는 트리거는 어떻게 할까요?

A) **자동 (권장)** — 사용자가 위쪽 끝 근처로 스크롤하면 다음 이전 청크가 자연스럽게
   나타남(위로 무한 스크롤). 별도 클릭 불필요.

B) **명시 버튼** — 목록 상단에 "이전 메시지 불러오기" 버튼을 두고, 누를 때마다 한 청크씩
   펼침. 동작이 더 예측 가능하고 스크롤 이벤트 처리 단순.

C) Other (please describe after [Answer]: tag below)

[Answer]: A

## Question 3
입력 영역 리디자인(FR-4) 범위는 어디까지 할까요?

A) **풀 리디자인 (권장)** — 상용 LLM 챗 스타일: 하나의 둥근 컨테이너 안에 textarea +
   전송/정지 버튼을 통합(우하단 아이콘 버튼), 내용에 따라 1→여러 줄 자동 확장(최대 높이 후
   내부 스크롤), 포커스 시 은은한 강조 링, 키보드 힌트/전송 상태 반영. 접근성·기존 키 동작 유지.

B) **경량 리스타일** — 지금 구조(별도 버튼)는 유지하고 여백/테두리/포커스만 다듬는 수준.
